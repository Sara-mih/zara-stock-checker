# Zara Stock Checker

Автоматски проверува дали производите од твоите Zara favorites се повторно достапни, и ти праќа email кога ќе стане достапно.

## Чекор по чекор

### 1. Направи нов приватен GitHub repo
На пример `zara-stock-checker`, и ги качуваш сите овие фајлови во него (може директно преку GitHub веб интерфејс - "Add file → Upload files").

### 2. Направи Gmail App Password
Ова е потребно за да можеш да праќаш email преку скрипта (не смееш да ја користиш обичната лозинка):
1. Оди на https://myaccount.google.com/apppasswords
2. Мора да имаш вклучено 2-Step Verification на Gmail сметката
3. Генерирај нов App Password (избери "Mail" / "Other")
4. Копирај го 16-цифрениот код

### 3. Додади ги Secrets во GitHub repo
Во repo → Settings → Secrets and variables → Actions → New repository secret:

| Name | Value |
|---|---|
| `GMAIL_USER` | твојата gmail адреса |
| `GMAIL_APP_PASSWORD` | 16-цифрениот код од чекор 2 |
| `NOTIFY_EMAIL` | каде сакаш да пристигне известувањето (може иста како GMAIL_USER) |

### 4. Пополни ги твоите производи во `products.json`
Отвори го фајлот и стави ги линковите од Zara (директно од продукт страницата, не од favorites - оти favourites бара логин):

```json
[
  {
    "name": "Кожен јакна",
    "url": "https://www.zara.com/mk/en/...-p12345678.html",
    "size": "M"
  },
  {
    "name": "Патики",
    "url": "https://www.zara.com/mk/en/...-p87654321.html",
    "size": "40"
  }
]
```

Ако сакаш да провериш дали производот воопшто е достапен во било која големина, само избриши го `"size"` полето.

### 5. Тоа е тоа
GitHub Actions автоматски ќе го пушта скриптот на секои 30 минути, дури и кога компјутерот ти е исклучен. Може рачно да го тестираш веднаш преку repo → Actions → "Zara Stock Check" → "Run workflow".

## Забелешки

- Zara понекогаш детектира и блокира автоматски барања (CAPTCHA). Ако забележиш дека статусот секогаш излегува "None" во логовите, значи структурата на страницата или заштитата се променила - тогаш јави се и ќе го прилагодиме селекторот.
- Секое известување се праќа само еднаш - штом производот стане достапен, state.json се ажурира за да не добиваш повторни emails за истата ствар (освен ако пак стане out-of-stock па повторно in-stock).
- Ако сакаш почесто проверување (пр. на секои 5 мин), промени го `cron` во workflow фајлот - но не претерувај, за да не те блокираат.
